# CFWEB 使用流程

本文档详细介绍 CFWEB 从配置到部署、从启动到打包的完整使用流程。

---

## 一、准备工作

### 1.1 确认依赖

CFWEB 仅依赖以下常见工具，请确保目标机器已安装：

- `bash`
- `curl`
- `jq`
- `python3`

检查命令：

```bash
bash --version
curl --version
jq --version
python3 --version
```

### 1.2 准备阿里云 RAM 用户

1. 登录 [阿里云 RAM 控制台](https://ram.console.aliyun.com/)。
2. 创建一个 RAM 用户，选择“OpenAPI 调用访问”。
3. 为该 RAM 用户授予 `AliyunDNSFullAccess` 策略（或自定义最小权限策略）。
4. 保存 AccessKey ID 和 AccessKey Secret。

### 1.3 准备 Cloudflare 账号

确保你有一个 Cloudflare 账号，并且你的域名（如 `example.com`）的 DNS 已托管在阿里云（因为脚本会调用阿里云 DNS API 添加 CNAME）。

---

## 二、配置 CFWEB

进入项目目录：

```bash
cd /AAAA/CFWEB
```

复制配置模板：

```bash
cp config.json.example config.json
```

编辑 `config.json`：

```bash
vim config.json
```

### 2.1 配置示例

```json
{
  "domain": "example.com",
  "tunnel_name": "cfweb-tunnel",
  "aliyun": {
    "access_key_id": "LTAIxxxxxxxxxxxx",
    "access_key_secret": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "region": "cn-hangzhou"
  },
  "services": [
    {
      "folder": "blog",
      "host": "localhost",
      "port": 3000,
      "description": "博客服务"
    },
    {
      "folder": "api",
      "host": "192.168.1.10",
      "port": 8080,
      "description": "内网 API 服务"
    }
  ]
}
```

### 2.2 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `domain` | 是 | 根域名，DNS 需托管在阿里云 |
| `tunnel_name` | 否 | Cloudflare Tunnel 名称，默认 `cfweb-tunnel` |
| `aliyun.access_key_id` | 是 | 阿里云 RAM 用户 AccessKey ID |
| `aliyun.access_key_secret` | 是 | 阿里云 RAM 用户 AccessKey Secret |
| `aliyun.region` | 否 | 阿里云区域，默认 `cn-hangzhou` |
| `services` | 是 | 需要暴露的服务列表 |
| `services[].folder` | 是 | 子域名前缀，仅支持英文、数字、连字符 |
| `services[].host` | 否 | 服务所在主机，默认 `localhost` |
| `services[].port` | 是 | 服务端口号 |
| `services[].description` | 否 | 服务说明 |

### 2.3 映射结果

以上配置会生成以下映射：

| 访问地址 | 目标服务 |
|----------|----------|
| `https://blog.example.com` | `http://localhost:3000` |
| `https://api.example.com` | `http://192.168.1.10:8080` |

## 三、一键安装

配置完成后，执行安装脚本：

```bash
./install.sh
```

脚本会自动完成以下步骤：

1. 检查 `curl`、`jq`、`python3` 是否已安装。
2. 下载适合你系统架构的 `cloudflared` 二进制。
3. 执行 `cloudflared tunnel login`，弹出浏览器授权页面。
   - 如果当前机器没有浏览器，请复制脚本输出的 URL 到本地浏览器完成授权。
4. 创建 Cloudflare Tunnel。
5. 生成 `config.yml` 配置文件。
6. 调用阿里云 DNS API，为每个 `folder` 添加/更新 CNAME 记录。
7. 启动 tunnel。

安装成功后，脚本会输出公网访问地址和管理命令。

## 四、管理 Tunnel

### 4.1 启动

```bash
./scripts/start-tunnel.sh
```

### 4.2 停止

```bash
./scripts/stop-tunnel.sh
```

### 4.3 查看状态

```bash
./scripts/status.sh
```

该命令会显示：

- tunnel 是否在运行
- 已配置的公网地址和本地目标
- 最近 20 行日志

## 五、配置 systemd 服务（可选）

如果你希望 tunnel 随系统启动而启动，可以使用 systemd：

```bash
CREATE_SERVICE=1 ./install.sh
```

安装完成后，使用 systemctl 管理：

```bash
sudo systemctl start cfweb
sudo systemctl stop cfweb
sudo systemctl restart cfweb
sudo systemctl status cfweb
sudo systemctl enable cfweb  # 开机自启
```

## 六、打包与跨机器部署

### 6.1 在开发/配置机器上打包

```bash
cd /AAAA/CFWEB
./scripts/package.sh
```

输出示例：

```text
打包完成: /tmp/cfweb-20260616_072937.tar.gz
```

打包文件会自动排除：

- `config.json`（包含敏感信息，需在新机器上重新填写）
- `config.yml`（安装时重新生成）
- `bin/cloudflared`（新机器重新下载）
- `credentials/` 下的凭证文件（新机器重新生成）
- 日志、PID、缓存文件

### 6.2 分发到其他机器

```bash
scp /tmp/cfweb-20260616_072937.tar.gz root@remote-host:/tmp/
```

### 6.3 在新机器上部署

```bash
ssh root@remote-host "
  cd /opt && \
  tar -xzf /tmp/cfweb-20260616_072937.tar.gz && \
  cd CFWEB && \
  cp config.json.example config.json && \
  vim config.json && \
  ./install.sh
"
```

新机器会重新下载 `cloudflared`、重新登录 Cloudflare、重新创建 tunnel、重新配置阿里云 DNS。

## 七、验证

### 7.1 检查 cloudflared

```bash
/AAAA/CFWEB/bin/cloudflared version
/AAAA/CFWEB/bin/cloudflared tunnel list
```

### 7.2 检查生成的配置

```bash
cat /AAAA/CFWEB/config.yml
```

### 7.3 检查 DNS 记录

等待 DNS 生效（通常几分钟），然后：

```bash
dig blog.example.com CNAME +short
```

应输出类似：

```text

```

```bash
curl -H "Host: blog.example.com" http://localhost:3000
```

### 7.5 公网测试

DNS 生效后：

```bash
curl https://blog.example.com
```

## 八、常见问题

### Q1: `cloudflared tunnel login` 无法弹出浏览器怎么办？

A: 脚本会输出一个授权 URL。请将该 URL 复制到本地有浏览器的机器上打开，完成授权后返回服务器继续。

### Q2: 阿里云 DNS 配置失败怎么办？

A: 检查：

- `config.json` 中的 `access_key_id` 和 `access_key_secret` 是否正确。
- RAM 用户是否具有 `AliyunDNSFullAccess` 权限。
- 域名是否已托管在阿里云 DNS。

### Q3: 打包时是否需要包含 `config.json`？

A: 不需要也不建议。`config.json` 包含敏感信息，打包脚本会自动排除。新机器部署时需重新填写。

### Q4: 能否把多个服务映射到同一个域名下的不同路径？

A: 当前版本仅支持子域名模式（`foldername.domain.tld`）。如需路径模式，请手动修改 `config.yml` 中的 `ingress` 规则。

### Q5: 服务不在本机怎么办？

A: 在 `config.json` 的服务配置中指定 `host` 字段，例如 `"host": "192.168.1.10"`。省略时默认使用 `localhost`。

## 九、安全提示

- `config.json` 包含阿里云 AccessKey Secret，请设置权限为 `600`。
- `credentials/` 目录包含 Cloudflare 凭证，请勿上传到公开仓库或随压缩包分发。
- 建议为 RAM 用户创建最小权限策略，仅允许对指定域名执行 DNS 操作。
- 定期更换 AccessKey Secret。
